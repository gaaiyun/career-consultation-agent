# 示例测试案例

这些案例来自当前产品最初定义时的典型职业咨询场景，适合直接复制到应用中测试四阶段工作流：

- `结构化拆解`
- `矛盾追问`
- `路线规划`
- `终版报告`

## 推荐测试顺序

1. `case_01_ruc_han_language.txt`
2. `case_02_music_major_transition.txt`
3. `case_03_b2b_content_operator.txt`
4. `case_04_ruc_labor_econ_gap.txt`

## 测试方法

### 基础测试

1. 在应用中新建案例
2. 复制对应 `.txt` 文件内容到“原始文本”
3. 依次运行四个阶段
4. 观察输出是否符合该案例的“预期观察点”

### 建议重点观察

- `结构化拆解` 是否能抓住核心矛盾，而不是只做摘要
- `矛盾追问` 是否真的在拆认知泡沫，而不是泛泛提问
- `路线规划` 是否能给出具体路径，而不是空泛方向
- `终版报告` 是否像一个成熟咨询师的回复，而不是普通 AI 总结

## 案例预期观察点

## Case 01: 人大汉语言大一学生

文件：`case_01_ruc_han_language.txt`

建议观察：

- 能否识别“求稳”与“转专业”之间的真实矛盾
- 能否把问题从“汉语言 vs 新闻”上升到“专业不动，技能动”
- 路线规划里是否能出现：
  - 高校行政/辅导员
  - 留京国央企
  - 人才引进/选调

## Case 02: 音乐学转行运营

文件：`case_02_music_major_transition.txt`

建议观察：

- 能否识别她不是“完全没能力”，而是“没有作品和具体方向”
- 追问是否会围绕“你理解的运营是什么”“能否接受音乐教培相关行业”
- 路线规划里是否能出现：
  - 音乐教培运营
  - 音乐平台运营
  - 会展/活动策划

## Case 03: 5年 B 端内容运营求职者

文件：`case_03_b2b_content_operator.txt`

建议观察：

- 能否识别“定位问题”大于“能力问题”
- 能否追问 B 端内容的真实商业逻辑
- 路线规划里是否能出现：
  - 去之前服务过的甲方
  - 具身智能内容运营
  - 大模型企业内容运营

## Case 04: 人大劳动经济学 GAP 求职者

文件：`case_04_ruc_labor_econ_gap.txt`

建议观察：

- 能否识别“市场没有识别她的能力”而不是简单归因为能力不够
- 能否追问金融岗位真实类型、GAP 解释、数分方向偏好
- 路线规划里是否能出现：
  - 内容出海/海外运营
  - 金融科技运营或分析
  - 游戏本地化/海外运营

## 可选扩展测试

你还可以用同一个案例做两轮测试：

1. 第一轮只输入原始文本
2. 第二轮把“追问补充回答”再手工填进去

这样更容易判断这套工作流是否真的具备“先拆解，再追问，再反推路线”的能力。
